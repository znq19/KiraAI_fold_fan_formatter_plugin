# 折扇留穗·格式转换器

> 将AI的长篇大论或特定标签内容，优雅地折叠成一张合并转发卡片，如同折扇收起时留下一缕穗子，聊天记录清爽有序。

---

## ✨ 插件简介

当 AI 输出过长消息或包含特定标签时，本插件会自动将消息转换为**合并转发卡片**，避免刷屏，同时保留引导语作为独立消息，形成「一行清语 + 一张精致卡片」的层次感。

- **自定义触发标签**：可自由定义起始和结束标签（例如 `[3p]`、`[dw]` 等）。
- **自动长度触发**：无需标签，超过设定字符数的消息也会自动转为卡片。
- **智能拆分**：将标签前的内容作为引导语单独发送，标签后的内容同样单独发送，卡片仅包含核心内容。
- **轻量高效**：仅依赖 KiraAI 原生能力，无需额外依赖。

---

## 🎯 功能特性

| 功能 | 说明 |
|------|------|
| **标签转换** | 识别 `[标签]内容[/标签]`，将内容转为合并转发卡片 |
| **长度触发** | 消息超过设定字符数（如 100 字）时自动转为卡片 |
| **引导语保留** | 标签前的文本作为普通消息先发送，形成“清语+卡片”效果 |
| **后续文本保留** | 标签后的文本也会单独发送，保证信息完整 |
| **仅群聊模式** | 可选择仅在群聊中生效，私聊保持原样 |
| **可配置标签** | 支持自定义起始/结束标签，适应不同格式需求 |

---

## 📦 安装

1. 将插件文件夹 `fold_fan_formatter` 放入 KiraAI 的 `data/plugins/` 目录。
2. 重启 KiraAI，插件会自动加载。
3. 在 WebUI 插件管理页面启用并配置（可选）。

---

## ⚙️ 配置项

通过 KiraAI 的 WebUI（插件管理 → 折扇留穗·格式转换器 → 配置）可修改以下选项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enabled` | switch | true | 插件总开关 |
| `only_group` | switch | false | 是否仅处理群聊消息（私聊不转换） |
| `tag_start` | string | `[3p]` | 起始标签，例如 `[3p]` 或 `[dw]` |
| `tag_end` | string | `[/3p]` | 结束标签，例如 `[/3p]` 或 `[/dw]` |
| `enable_auto_by_length` | switch | false | 是否开启字符数自动触发 |
| `auto_length_threshold` | integer | 100 | 触发自动转换的字符数阈值（仅当开启自动触发时有效） |

> **提示**：标签和长度触发可同时开启，插件会优先匹配标签；若未匹配标签但超过长度阈值，也会转为卡片。

## 🧩 使用示例

### 标准格式（推荐）

```xml
<msg>
    <text>此处写引导语（不包含标签）</text>
    <text>[标签]
此处写需要折叠的核心内容
[/标签]</text>
</msg>
```

插件处理后：
1. **引导语**（普通消息）：`此处写引导语`
2. **合并转发卡片**（点击查看）：核心内容

### 仅使用长度触发

AI 输出一条超过 100 字的纯文本消息（无标签），插件自动将其转为合并转发卡片。字符数可配置项中自定义。

## ✍️ 书写要诀（AI 提示词模板）

为了让插件完美工作，请务必遵循以下格式规范：

- **引导语（穗）**：写在标签（如 `[3p]`）之前，作为独立 `<text>` 元素。它是 AI 本人的直接回应，不依赖标签内的内容来解释自己。
- **折叠内容（扇面）**：标签内部是需要折叠的长文本，可以是分析、引用、多角度观点等。
- **标签必须被 `<text>` 元素包裹**，否则解析时会丢失。

### 通用提示词模板（可直接放入 KiraAI 人设相应部分）

```text
## 输出格式规范（折扇留穗）

当你需要输出较长的结构化内容时，请使用以下格式将其折叠：

<msg>
    <text>【短引导语】简要回应或开场白（不含标签）</text>
    <text>[标签]
// 这里放置需要折叠的长内容，可以是多行文本、列表、引用等
[/标签]</text>
</msg>

注意事项：
- 引导语与标签必须分属两个不同的 <text> 元素。
- 标签内的所有内容将被合并为一条可点击查看的卡片消息。
- 如果消息本身较短（小于阈值），可以不使用此格式，直接发送。
- 标签名称可自定义（插件默认 [3p]），请保持一致。
```

将此提示词添加到 KiraAI 的人设文件中，AI 就会按标准格式输出，插件即可识别。

---

## ⚠️ 注意事项

- **标签必须位于 `<text>` 元素内部**，否则 KiraAI 解析时可能丢失。请确保 AI 输出格式规范。
- **合并转发卡片内的内容会折叠**，用户需点击才能查看全部，适合长文或结构化内容。
- **延迟顺序**：引导语与卡片之间设有微小延迟（0.1秒），但受网络影响，偶尔卡片可能先出现，属正常现象。
- **仅限 QQ 适配器**：本插件依赖 OneBot 协议的 `send_forward_msg` 接口，仅支持 QQ 适配器（NapCat/LLOneBot）。
- **私聊兼容**：私聊同样支持合并转发。

## 🛠️ 常见问题

**Q：为什么引导语和卡片顺序偶尔颠倒？**  
A：这是由于网络延迟、服务器处理顺序或AI格式理解有限导致，插件已增加延迟，但仍无法100%保证。若严重影响使用，可将引导语也纳入标签内部（即标签包含全部内容）。

**Q：标签没被识别，消息原样发送？**  
A：请检查标签格式是否正确（起始和结束标签匹配），且位于 `<text>` 内部。同时确保插件未禁用，且 `only_group` 配置与当前场景一致。

**Q：字符数自动触发不生效？**  
A：请确认 `enable_auto_by_length` 为 `true`，且消息文本长度（不含标签）确实超过 `auto_length_threshold`。

**Q：能否在同一个消息中使用多种标签？**  
A：目前插件只处理一种设定好的匹配标签，其余原样发送。
